# Clean Code Review: dddns (full codebase)

**Date**: 2026-04-17
**Files reviewed**: 23 (21 production, 2 skipped trivial — `internal/constants/permissions.go`, `internal/version/version.go`)
**Findings**: 18 total — 1 critical, 7 refactor, 10 minor

## Executive summary

The codebase is in good health for its stated scope: tight, readable, idiomatic Go, with security-first defaults (fail-closed serve-mode, constant-time auth, atomic status writes, scoped context timeouts). The main weaknesses are a pocket of stale dead code in `internal/config/` (the `OnAuthFailure` field, never read anywhere), some legacy cruft around the `cobra.Command` init wizard (over-long function, package-level state for `forceInit`/`interactive`), and a documentation-vs-code mismatch around `dddns secure disable`. `cmd/config.go`'s `runInteractiveConfig` is the one function that clearly deserves a refactor pass. No critical bugs were found.

## Positives

- Security posture is consistently fail-closed: `ServerConfig.Validate` rejects empty allowlists, `ServerConfig.Validate` rejects empty secrets, `auth.go` uses `subtle.ConstantTimeCompare`, `status.go` uses write-to-temp+rename for atomicity, audit log writes append-only with O_APPEND.
- `internal/updater/updater.go` cleanly extracts the shared flow between cron (`cmd/update.go`) and serve (`internal/server/handler.go`). `DNSClient` being a 2-method interface is right-sized for the current codebase.
- Context propagation is tight in the code paths that matter: `updater.Update(ctx, ...)`, `Route53Client.GetCurrentIP(ctx)`, `Route53Client.UpdateIP(ctx, ...)`, `handler.ServeHTTP` bounded with `handlerTimeout`.
- Error wrapping is universally `%w` — no `return err` boundary losses detected.
- `internal/server/audit.go` comments explicitly call out the single-`os.File.Write`-is-atomic-below-PIPE_BUF invariant — that's real operational awareness.
- `internal/wanip/wanip.go`'s test seam (`interfaceAddrs` func var, `defaultRoutePath` string var) is clean and minimal.
- The boot-script generator (`internal/bootscript/bootscript.go`) is pure string templating with no side effects — ideal shape for a unit-testable platform glue module.
- `cmd/config_rotate_secret.go:92` explicitly acknowledges audit-write failure must not block printing the new secret. That's a thoughtful ordering.

## Recommendations

### Fix Now (critical)

- **Dead code #1 (`OnAuthFailure`)**. It's carried through the secure-config encryption path, so the field is now part of the on-disk schema with no functional behaviour. Either wire it or delete it before more `.secure` files are written with the field present.

### Defer to Issues (refactor)

- **Complexity #1** — `GetDeviceKey` refactor (cleanest win for readability).
- **SRP #1 + Complexity #2** — `runInteractiveConfig` decomposition.
- **Coupling #1** — drop `profile.Current`, return the profile from `Detect`.
- **Ignored errors #3** — `MigrateToSecure` should surface wipe errors.
- **Dead code #3** — reconcile `UpdateIP(dryRun)` vs. `updater.Update` short-circuit.
- **Go-specific / missing context #2** — plumb `ctx` into `myip.GetPublicIP`.
- **Doc drift** — `.claude/CLAUDE.md:117` advertises `dddns secure disable`, which is not implemented. Either implement or remove from the doc.

### Skip (trivial/contentious)

- Naming #2 (package-level cobra flag vars) — contentious; the pattern is widespread in cobra projects.
- Dead code #4 (`GetBuildDate`) — file is tiny; not worth a PR.
- `ValidatePublicIP` accepts CGNAT while `isPublicIPv4` rejects it — seemingly inconsistent but actually correct for each caller's context (remote-check-from-internet vs. my-own-WAN-interface). Not a finding.
